## Техническая документация
### Описание проекта
API для сервиса [https://skillcrafts.ru](https://skillcrafts.ru)
с JWT аутентификацией (пользователи + гости). Чтобы познакомиться с ресурсом можно войти под учётной записью `user@example.com` с паролем `userUSER1!`. Все изменения, совершаемые под этой учётной записью, восстанавливаются в исходное состояние каждый час.

Подробнее о сервисах проекта ниже. Перейти к ендпоинтам
[**endpoints**](/api/docs/#/schema).

#### Сервис **auth**
Данный сервис работает с JWT токенами для доступа к ресурсу. Что такое JWT можно подробнее изучить [на хабре](https://habr.com/ru/articles/842056/)

Больше подробностей в ендпоинтах [**auth**](/api/docs/#/users).

#### Сервис **users**
Данный сервис проводит регистрацию пользователей и контролирует их вход.
Проводит аутентификацию и разграничение прав доступа.

Доступ к сервисам проекта можно получить как под временной учётной записью (гостем),
так и под постоянной. Для постоянной учётной записи при регистрации необходимо
указать и подтвердить **email**.

Больше подробностей в ендпоинтах [**users**](/api/docs/#/users).

#### Сервис **privacy-settings**
Данный сервис позволяет пользователю гибко управлять доступом к своему контенту, реализуя политику конфеденциальности [**policy**](/api/docs/#/policy).

В этом сервисе настраивается чёрный и белый списки пользователей, а также тонко настраивать видимость персональных данных, указанных в своём профиле (см. следующий сервис **profiles**).

Всего можно настроить 4 уровня доступа к персональному контенту:
1. Видно всем;
2. Видно всем, кроме...;
3. Невидно никому, кроме...;
4. Невидно никому.

Указанная градация уровней применима к использованию в конкретном контексте.
Больше подробностей в ендпоинтах [**privacy-settings**](/api/docs/#/privacy-settings),
а также во всех ендпоинтах, контент которых содержит поле `privacy`.

#### Сервис **profiles**
Данный сервис позволяет гибко настраивать профиль пользователя. Указывать имя, фамилию, местонахождение, уровень образования и другие персональные данные. Также можно загрузить аватар и обои для профиля.

Для каждого поля профиля можно установить политику конфиденциальности (например, скрыть свою локацию от всех или только от тех, кто в чёрном списке). Это же касается и изображений. Ни одно изображение, загруженное пользователем, недоступно неавторизованному пользователю (анониму).

В профиле можно настроить стратегию политики конфиденциальности:
- От максимальной открытости (открытый профиль);
- До максимальная конфиденциальности (закрытый профиль).

В профиле есть возможность гибко настроить стратегию для каждого сервиса.

Больше подробностей в ендпоинтах [**profiles**](/api/docs/#/profiles).

#### Сервис **resume**
Данный сервис позволяет создавать одно резюме. Сервис помогает формулировать и структурировать полученный опыт.

С помощью сервиса можно создать как классическое резюме, так и прогрессивное. В резюме есть все артефакты для этого. В резюме можно добавлять, редактировать, удалять или скрывать места работы или их подробности.

Для полей резюме также предусмотрена настройка конфиденциальности через поле контента `privacy`. Значение по умолчанию устанавливается в соответствии с выбранной стратегией из настроек профиля.

Больше подробностей в ендпоинтах [**resume**](/api/docs/#/resume).

#### Сервис **workflows**
Данный сервис позволяет полноценно вести проекты.

С помощью сервиса пользователь может создавать, редактировать, удалять проекты и задачи. Двигать задачи по рабочему процессу (Очередь - Готовность к работе - В работе - На проверке - Выполнено/Отменено/Удалено). Привязывать задачи к проектам, отмечать задачи тегами и фильтровать задачи. Списывать на задачи время.

Также для задач можно устанавливать сроки и приоритеты, видеть историю изменения задачи и минимальную статистику (время нахождения на каждом этапе рабочего процесса).

Для задач также предусмотрена настройка конфиденциальности через поле контента `privacy`. Значение по умолчанию устанавливается в соответствии с выбранной стратегией из настроек профиля.

Больше подробностей в ендпоинтах [**workflows**](/api/docs/#/workflows).

#### Сервис знаний (**knowledge**)
Данный сервис позволяет пользователю составлять карту своих знаний в виде
дерева заметок (доступна опция форматированного ввода markdown).

По своей структуре это классическая карта знаний (MindMap). Пользователь по своему желанию, может составлять, редактировать и удалять карты, а также иметь лёгкий доступ к ним. Пользователь может просматривать чужие карты знаний, если они публичные или к ним есть доступ.

Пользователь может гибко управлять видимостью своих знаний. Каждая заметка имеет поле `privacy` со значением по умолчанию:
- **видно всем** для стратегии открытого профиля;
- **невидно никому** для стратегии закрытого профиля.

Больше подробностей в ендпоинтах
[**knowledge**](/api/docs/#/knowledge).
